1.Что такое сложность алгоритма
Сложность показывает, как изменяется количество операций или расход памяти при увеличении объёма входных данных.
Обычно используют:
    - временную сложность — сколько операций выполняется;
    - пространственную сложность — сколько дополнительной памяти требуется.
Пока рассматриваем временную сложность. Объём входных данных обозначают буквой n.
Например:
в массиве из 10 элементов n = 10;
в массиве из 1000 элементов n = 1000;
в строке из 50 символов n = 50.

2. Что означает Big O
Big O показывает, с какой скоростью растёт количество операций при увеличении n.
Важно: Big O обычно не показывает точное количество операций. Она описывает характер роста.
Например, алгоритм выполняет:
3n + 5 операций.
При большом n главную роль играет n, поэтому сложность записывают: O(n)
Константы и менее значимые части отбрасываются:
    • 5 → O(1)
    • 2n + 10 → O(n)
    • n² + 5n + 20 → O(n²)
    • 3n log n + n → O(n log n)

3. Основные сложности
O(1) — постоянная сложность. Количество операций не зависит от размера коллекции.
Пример: получить элемент массива по известному индексу.
Неважно, в массиве 10 элементов или 10 миллионов — обращение к конкретному элементу выполняется примерно за одно действие.
Условно: Количество операций = 1

O(log n) — логарифмическая сложность
На каждом шаге объём данных значительно уменьшается, обычно в два раза.
Главный пример — бинарный поиск. Допустим, есть 1000 элементов:
    • после первого шага остаётся примерно 500;
    • затем 250;
    • затем 125;
    • затем 62;
    • и так далее.
Для 1000 элементов потребуется примерно 10 шагов, потому что: 2¹⁰ = 1024. Это очень эффективная сложность.

O(n) — линейная сложность
Количество операций растёт пропорционально количеству элементов. Пример: поиск значения последовательным просмотром массива.
Если элементов:
    • 10 — максимум около 10 проверок;
    • 100 — максимум около 100 проверок;
    • 1 000 — максимум около 1 000 проверок.
Если размер данных увеличился в два раза, количество операций тоже примерно увеличится в два раза.

O(n log n) — линейно-логарифмическая сложность
Эта сложность часто встречается в эффективных алгоритмах сортировки:
    • быстрая сортировка — в среднем;
    • сортировка слиянием;
    • пирамидальная сортировка.
Она хуже, чем O(n), но значительно лучше, чем O(n²). Например, при n = 1000:
    • O(n) — примерно 1 000 операций;
    • O(n log n) — примерно 10 000 операций;
    • O(n²) — примерно 1 000 000 операций.
Это условные значения, но они хорошо показывают разницу.

O(n²) — квадратичная сложность. Обычно возникает, когда для каждого элемента перебираются остальные элементы.
Например:
    • внешний перебор проходит n элементов;
    • для каждого элемента внутренний перебор также проходит n элементов.
Получается: n × n = n²
При увеличении данных в два раза количество операций увеличивается примерно в четыре раза:
    • n = 10 → 100 операций;
    • n = 100 → 10 000 операций;
    • n = 1000 → 1 000 000 операций.
Такая сложность характерна для простых сортировок:
    • пузырьковой;
    • выбором;
    • вставками — в среднем и худшем случае.

4. Сравнение
От лучшей к худшей: O(1) → O(log n) → O(n) → O(n log n) → O(n²)
При n = 1 000 приблизительно:
| Сложность  | Количество операций |
| ---------- | ------------------: |
| O(1)       |                   1 |
| O(log n)   |                  10 |
| O(n)       |               1 000 |
| O(n log n) |              10 000 |
| O(n²)      |           1 000 000 |

5. Что отбрасывается при вычислении сложности
Константы.
O(2n) превращается в O(n).
Причина: важно не точное количество операций, а скорость роста.
Оба варианта растут линейно:
    • n = 10: n = 10, 2n = 20;
    • n = 1000: n = 1000, 2n = 2000.
2n больше, но характер роста остаётся линейным.

Менее значимые части
O(n² + n) превращается в O(n²).
При больших значениях n квадрат растёт значительно быстрее:
    • при n = 10: n² = 100, n = 10;
    • при n = 1000: n² = 1 000 000, n = 1000.
Поэтому n становится несущественным на фоне n².

6. Последовательные и вложенные действия
Если сначала перебирается один массив, а затем другой:
O(n) + O(n) = O(2n) = O(n)
Последовательные действия складываются.
Если один перебор находится внутри другого:
O(n) × O(n) = O(n²)
Вложенные действия умножаются.

7. Лучший, средний и худший случаи
У одного алгоритма сложность может зависеть от ситуации.
Например, линейный поиск:
     • искомый элемент первый — O(1);
     • элемент находится примерно посередине — в среднем O(n);
     • элемент последний или отсутствует — O(n).
Когда сложность указывают без уточнения, чаще всего имеют в виду худший случай.

Главное.
Сложность — это не точное время в секундах. Она показывает, как растёт объём работы алгоритма при росте входных данных.
Нужно запомнить:
     • O(1) — количество данных не влияет;
     • O(log n) — данные на каждом шаге сокращаются;
     • O(n) — один полный перебор;
     • O(n log n) — эффективные сортировки;
     • O(n²) — вложенный перебор.
